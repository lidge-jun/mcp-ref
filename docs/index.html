Policy before tool sprawl.

Include when the CLI cannot faithfully replace it.

Good MCP servers reach authenticated external systems, expose live state, compress huge API surfaces, or provide structural intelligence that plain shell commands cannot recreate.

Reject duplicate wrappers.

Filesystem, Git, basic GitHub, fetch-only, and memory-only wrappers add noise when the agent already has better native tools.

Keep credentials out.

Registry entries use placeholders such as YOUR_TOKEN. Real secrets belong in user-level configuration.

Use one entry at a time.

Most workflows need only a few MCP servers. Add one, prove it is useful, then add another. This keeps context and failure modes legible.

Verification is simple and static.

The repo is a JSON registry plus static documentation. CI validates JSON shape, count drift, local links, Pages metadata, and whitespace hygiene before deploy.

jq empty registry.json
jq -e '.version == 3 and (.servers | length == 13)' registry.json
git diff --check
